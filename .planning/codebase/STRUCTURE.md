# Codebase Structure

**Analysis Date:** 2026-07-08

**Status note:** This repository is in an early/design stage. There is no `src/` package, no build system, no test directory, and no dependency manifest (`requirements.txt`/`pyproject.toml`) yet — dependencies are installed ad hoc by `.devcontainer/post-create.sh`. This document describes the structure that exists today and flags where the design doc implies future structure that has not been created.

## Directory Layout

```
enpipe/
├── .devcontainer/           # Dev environment definition (Docker + VS Code devcontainer)
│   ├── Dockerfile            # Intel Arc QSV media toolchain image (ffmpeg, qsvencc, dovi_tool, mkvtoolnix)
│   ├── devcontainer.json     # Container config: GPU passthrough, media folder mounts, features
│   ├── devcontainer-lock.json # Pinned feature versions (auto-generated by devcontainer CLI)
│   └── post-create.sh        # Post-create provisioning: GPU group perms, npm CLIs, pip deps, self-check
├── .planning/                # GSD planning artifacts (this document lives here)
│   └── codebase/              # Generated codebase-map documents (ARCHITECTURE.md, STRUCTURE.md, ...)
├── legacy/                   # Prior-project Python scripts, kept for reference/reuse
│   ├── scene_detection.py     # Stage 1: GPU-accelerated scene-cut detection CLI
│   └── encode_scenes.py       # Stage 2: scene-aware AV1 chunked encoding CLI
├── .gitignore                 # Standard Python .gitignore (byte-compiled files, venvs, coverage, etc.)
├── LICENSE                    # Project license
└── PIPELINE_DESIGN.md         # Design doc (Russian) for a proposed streaming detect→encode pipeline; NOT implemented
```

There is currently no directory for: application source code (`src/`), tests (`tests/`), scripts/tooling beyond the devcontainer, configuration files (beyond the devcontainer), or CI (no `.github/workflows/`).

## Directory Purposes

**`.devcontainer/`:**
- Purpose: Defines the reproducible development/runtime environment for this GPU-media project — a Debian trixie-based container with Intel Arc QSV/VA-API drivers, `ffmpeg`, `qsvencc` (Rigaya AV1 encoder), `mkvtoolnix`, and `dovi_tool` preinstalled, plus Node-based AI CLI tooling (Claude Code, opencode, qwen-code).
- Contains: `Dockerfile` (image build), `devcontainer.json` (container runtime config — GPU device passthrough via `--device=/dev/dri`, host media folder bind mounts `/data/media` and `/data/downloads`), `post-create.sh` (runtime provisioning script), `devcontainer-lock.json` (feature version lock, auto-generated).
- Key files: `.devcontainer/Dockerfile` for build-time toolchain; `.devcontainer/post-create.sh` for one-time runtime setup (GPU group membership, `pip install scenedetect[opencv-headless] numpy`, environment self-check printing tool versions).

**`legacy/`:**
- Purpose: Holds the two working Python scripts from a prior iteration of this project, treated as the functional baseline that any new orchestrator/pipeline code should build on or replace.
- Contains: Two standalone, no-external-project-dependency Python files (each is directly runnable via `python3 legacy/<script>.py`).
- Key files: `legacy/scene_detection.py` (PySceneDetect + QSV-decode-pipe scene boundary detection, writes `<video>.scenes`), `legacy/encode_scenes.py` (reads `<video>.scenes`, produces chunked AV1 encode + final `mkvmerge` mux).

**`.planning/`:**
- Purpose: GSD (Get-Stuff-Done) planning workflow artifacts — codebase maps, phase plans, etc., generated/consumed by `/gsd:*` slash commands.
- Contains: `codebase/` subdirectory with generated `*.md` reference documents (this document is one of them).
- Generated: Yes (by mapper agents). Committed: typically yes, to keep planning context versioned with the repo.

## Key File Locations

**Entry Points:**
- `legacy/scene_detection.py`: Scene-detection CLI (`python3 legacy/scene_detection.py <input.mkv> [--jobs N] [--width W] ...`), `__main__` block at line 647.
- `legacy/encode_scenes.py`: Encoding CLI (`python3 legacy/encode_scenes.py <video> <scenes-log> [--jobs N] ...`), `main()` at line 515, `__main__` guard at line 727.

**Configuration:**
- `.devcontainer/devcontainer.json`: Dev environment configuration (GPU device passthrough, mounts, features, post-create hook).
- `.devcontainer/Dockerfile`: Build-time toolchain configuration/versions.
- No project-level runtime config file exists; both legacy scripts are configured via CLI flags and environment variables (`ICQ`, `QPMAX`, `GOP_LEN`, `DV_PROFILE`, `JOBS`, `FLAC_LEVEL`, `AUDIO_COPY` for `encode_scenes.py`; see `legacy/encode_scenes.py:52-59`).

**Core Logic:**
- `legacy/scene_detection.py`: All scene-detection logic (ffprobe wrapper, `QsvPipeStream`, sequential and parallel detection, CLI).
- `legacy/encode_scenes.py`: All encoding orchestration logic (keyframe table, HDR/DV detection, chunk building, threaded encode + ordered append, audio, mux).

**Design Reference:**
- `PIPELINE_DESIGN.md`: Proposed unified streaming architecture, cost/benefit analysis, and explicit "not implemented" status for a fused detect+encode pipeline. Read this before starting any phase that touches pipeline orchestration — it documents a considered-and-rejected-for-now approach with concrete numbers, and outlines the specific functions (`detect_scenes_streaming`, encoder `main()` refactor) that a future implementation phase would need to add.

**Testing:**
- None present. No `tests/` directory, no `pytest.ini`/`pyproject.toml` test config, no CI workflow.

## Naming Conventions

**Files:**
- Python modules: `snake_case.py` matching their primary responsibility as a verb phrase (`scene_detection.py`, `encode_scenes.py`).
- Design/reference docs at repo root: `SCREAMING_SNAKE_CASE.md` (`PIPELINE_DESIGN.md`, `LICENSE`).
- Generated intermediate artifacts follow the pattern `<source-stem>.<suffix>`: e.g. `movie.mkv` → `movie.mkv.scenes` (scene log), `<out>.metrics.csv` (per-scene metrics CSV), `<out>.chunks/` (chunk work directory), `chunk_00007.obu` (zero-padded 5-digit chunk index).

**Directories:**
- Top-level directories are lowercase, single-word or dot-prefixed for tooling (`legacy/`, `.devcontainer/`, `.planning/`).
- Work directories created at runtime by `encode_scenes.py` follow `<output-stem>.chunks/` (default `args.workdir`, `legacy/encode_scenes.py:539`).

**Python code conventions observed in `legacy/`:**
- `snake_case` for functions and variables, `PascalCase` for classes/dataclasses (`DetectionConfig`, `SourceInfo`, `Scene`, `QsvPipeStream`).
- Frozen dataclasses (`@dataclass(frozen=True)`) for value objects passed through the pipeline.
- Module-level `UPPER_CASE` constants for tunables sourced from environment variables (`ICQ`, `QPMAX`, `GOP_LEN`, `DV_PROFILE`, `JOBS`, `FLAC_LEVEL`, `legacy/encode_scenes.py:52-57`).
- Leading-underscore prefix (`_detect_relative`, `_boundary_worker`, `_sanitize_boundaries`, `_ebml_num`) marks internal/private helper functions not part of the intended external CLI/API surface.
- Extensive Russian-language docstrings and inline comments throughout both legacy scripts and `PIPELINE_DESIGN.md`.

## Where to Add New Code

**New pipeline/orchestrator code (per `PIPELINE_DESIGN.md`):**
- If a streaming/pipelined orchestrator is implemented, `PIPELINE_DESIGN.md` specifies it should add `detect_scenes_streaming()` "next to `detect_scenes`" — i.e., inside `legacy/scene_detection.py` (or its eventual replacement module) — and refactor `encode_scenes.py`'s `main()` (current lines 542-645) so its scene-source is queue-fed rather than list-fed, with the batch path becoming a special case of the streaming path. No project package structure (`src/`) exists yet for this to live in; establishing one (e.g. `src/enpipe/`) would be a reasonable first step for any phase that goes beyond quick script edits.

**New Feature / New Module:**
- Given the current flat two-script layout, any new stage (e.g. a real orchestrator, a config-file loader, a test suite) should introduce a proper package directory (e.g. `src/` or a top-level package matching the repo name) rather than adding more standalone top-level scripts, to avoid repeating the "legacy" pattern of unconnected CLI scripts glued together only by intermediate files.
- Tests: no existing test directory or framework choice has been made; a new `tests/` directory with `pytest` (already a natural fit given `pip install scenedetect[opencv-headless] numpy` in `post-create.sh`) would need to be introduced from scratch.

**Utilities / Shared helpers:**
- None currently extracted — both legacy scripts are self-contained with no shared internal module. If code becomes shared between a detection module and an encoding module (e.g. a common `ffprobe` wrapper, a common logging helper), it should be factored into a new shared module rather than duplicated, since `probe_source`/`probe_fps` and various ffprobe-invocation patterns are currently duplicated with different scopes between the two files.

**Devcontainer/toolchain changes:**
- New system package or binary tool installs belong in `.devcontainer/Dockerfile` (build-time) or `.devcontainer/post-create.sh` (runtime provisioning, e.g. new pip packages) — keep the split consistent with existing usage: Dockerfile for anything requiring root/apt, post-create.sh for user-level setup and idempotent checks.

## Special Directories

**`legacy/`:**
- Purpose: Explicitly named to signal this is prior/inherited code, not the current canonical implementation — treat it as a reference baseline to migrate from, not a location for new feature work.
- Generated: No (hand-written).
- Committed: Yes.

**`.planning/codebase/`:**
- Purpose: Auto-generated codebase reference documentation consumed by GSD planning/execution commands (`/gsd:plan-phase`, `/gsd:execute-phase`).
- Generated: Yes.
- Committed: Yes (recommended, to keep planning context in version control).

**Runtime work directories (not present in repo, created at run time by `encode_scenes.py`):**
- `<out>.chunks/`: Holds per-scene `.obu` chunk files, `movie.obu` (concatenated raw AV1 stream), and `audio.mka` during a run; deleted at the end unless `--keep` is passed (`legacy/encode_scenes.py:539-540`, `:708-716`). Not committed, not present until a run occurs; not currently covered by `.gitignore` by name (would need a glob like `*.chunks/` added if runs happen inside the repo tree — in practice this is expected to run against files under `/data/media`/`/data/downloads`, mounted outside the repo per `.devcontainer/devcontainer.json`).
