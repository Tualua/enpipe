<objective>
Add a third `enpipe` subcommand, `enpipe run <video>`, that composes the already-verified `run_detect` -> `run_encode` sequentially in a single invocation and produces the final `.mkv` — a thin orchestrator with zero behavior change to either stage.

Purpose: Close the "one command" UX gap (RUN-01..RUN-04) at zero regression risk. Byte-identical parity to the manual two-step is guaranteed by construction (same Namespace values, same `<video>.scenes` on disk); the tests prove it. An additive fail-fast preflight makes `enpipe run` die before wasting a (potentially long) detect pass when an encode-stage tool is missing.
Output: A `run` subparser + orchestrator handler in `cli/main.py`, a fast mocked unit test, and a hardware-gated parity test.
</objective>
