<objective>
Capture the mandatory parallel==sequential regression baseline (TEST-03) against the DEBT-03-resolved implementation.

Purpose: This test is the guard that DEBT-03's executor resolution did not change detection output, and the standing guard against any future regression in the multi-segment stitching path (`_sanitize_boundaries`, `non_cut_offsets` merge). Per D-05/D-06 it uses the software-decode fallback so it runs in ordinary CI without GPU. The single load-bearing risk (verified in RESEARCH): `detect_scenes_parallel` silently falls back to sequential unless `total_frames >= jobs * min_span` (min_span = max(2*min_scene_len, round(60*fps)) = 1440 frames = 60s at 24fps). A too-short clip makes the equality test vacuous — so this plan adds BOTH a pure gate-arithmetic unit test AND the real-clip regression test.

Output: Two committed tests in the default (non-hardware) tier.
</objective>
