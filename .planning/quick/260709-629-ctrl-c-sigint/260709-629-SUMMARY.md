# Quick Task 260709-629: Ctrl-C / SIGINT Summary

**`enpipe` теперь тихо завершается по Ctrl-C: короткое русское сообщение в stderr и код выхода 130 вместо необработанного Python-трейсбека из main()/ThreadPoolExecutor.**

## Accomplishments
- `main()` в `src/enpipe/cli/main.py` оборачивает `parser.parse_args(argv)` + `args.func(args)` в `try/except KeyboardInterrupt`, печатает `"enpipe: прервано (Ctrl-C)"` в stderr и поднимает `SystemExit(130)`
- Новый быстрый юнит-тест `tests/unit/cli/test_cli_sigint.py` доказывает поведение через monkeypatch `run_detect`, бросающий `KeyboardInterrupt`, и проверку `SystemExit.code == 130`
- `SystemExit` от `parse_args` (например, при отсутствии subcommand) по-прежнему пробрасывается нормально — перехватывается только `KeyboardInterrupt`

## Files Created/Modified

- `tests/unit/cli/test_cli_sigint.py` - новый юнит-тест по образцу `tests/unit/cli/test_cli_dispatch.py`

## Decisions Made
- Перехват только `KeyboardInterrupt` на верхнем уровне `main()`; без `signal.signal`, без вмешательства в `ThreadPoolExecutor` или дочерние процессы — они гибнут сами по групповому SIGINT при разворачивании стека (локированное решение из плана)
- `raise SystemExit(130)` с целочисленным кодом, сообщение печатается отдельным `print(..., file=sys.stderr)` перед этим, чтобы код `130` не попал в stderr как текст
