# Quick Task 260709-4h8: `-o` accepts a directory

## Objective

When the `-o/--out` value is an EXISTING directory, the output file is named `<original video stem>.Encoded<original suffix>` inside it (e.g. `movie.mkv` → `<dir>/movie.Encoded.mkv`), and the chunks workdir then also lands inside that directory (fixes the `PermissionError` a user hit passing a directory to `-o`). Otherwise `-o` stays the exact output file path, and omitting `-o` keeps the current default (`<video>.av1.mkv`). Shared by both `enpipe encode` and `enpipe run` (both call `run_encode`).

## Locked decisions
- Directory detection = `Path.is_dir()` (an EXISTING directory). argparse `type=Path` strips trailing slashes, so trailing-slash intent on a non-existent path is not detectable — existing-dir is the reliable, least-surprising rule.
- ZERO behavior change for `-o <file>` and omitted `-o` (preserves the byte-identical hardware parity test, which passes explicit file paths).
- Preserve conventions: Russian docstrings/help, `typing`-module generics (Optional/…) NOT PEP 604, `# --- … --- #` banners.
- Do NOT touch `legacy/`, do NOT change encode algorithm behavior.
