<objective>
Дать `enpipe run`, `enpipe detect`, `enpipe encode` возможность принимать
директорию и пакетно обрабатывать все видеофайлы внутри (рекурсивно по
`--recursive`), сохраняя collect-then-report семантику проекта: продолжать при
ошибке отдельного файла, пропускать уже готовые, в конце печатать сводку и
падать ненулевым кодом при наличии сбоев.

Purpose: убрать ручной цикл по файлам для NAS-батчей; частый реальный сценарий —
«прогнать всю папку сезона».
Output: новый leaf-модуль `shared/batch.py` (дискавери + оркестратор) и
диспетчеризация папка/файл в трёх подкомандах; поведение ОДИНОЧНОГО файла
байт-идентично прежнему.
</objective>
